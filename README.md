# Volleyball Training Bot (Telegram)

Телеграм-бот для записи на волейбольные тренировки через **сообщения каналу** (Channel Direct Messages) и управления записью из личного чата админами.

- Админ открывает запись командой `/new_training` → бот присылает администратору **готовый текст поста**, чтобы он опубликовал его в канал вручную (и при желании поставил **отложенную публикацию** прямо в Telegram).
- Пользователи пишут в сообщения каналу: `Секция ФИО` → бот записывает в основной список или в лист ожидания.
- Команда `Отмена` в сообщениях каналу снимает запись и автоматически поднимает первого из ожидания.
- Всё хранится в SQLite (`volleyball_bot.db`).

## Возможности

### Для пользователей (в сообщениях каналу)

Доступные сообщения/команды:

- `Секция ФИО` — записаться (пример: `Секция Иванов Иван Иванович`)
- `Инфо` — информация по текущей тренировке и занятости мест
- `Мой номер` — ваш статус и номер в очереди
- `Отмена` — отменить запись (с автопереводом из ожидания)

### Для админов (в личных сообщениях с ботом)

- `/new_training` — пошагово открыть запись (дата/время/лимит/уровень → **время публикации поста** → подтверждение)
- `/close_training` — закрыть текущую запись
- `/training` — показать текущую тренировку и счётчики
- `/list` — основной список участников
- `/waiting` — лист ожидания
- `/edit_training` — изменить лимит мест у открытой тренировки
- `/admins` — список админов
- `/cancel` — отмена пошагового ввода
- `/my_id` — показать ваш `user_id`

### Публикация поста (вручную, можно с отложкой)

При создании новой тренировки бот спрашивает: **когда вы планируете опубликовать пост** (по Москве) — это нужно для информации.

Важно: бот **не публикует** посты в канал автоматически. Он отправляет вам текст поста в личные сообщения — вы копируете его и публикуете/планируете публикацию в Telegram.

Варианты ввода:
- `сразу` — публикация поста в канал сразу после подтверждения
- `ДД.ММ.ГГГГ ЧЧ:ММ` — публикация в указанное время **по Москве**

Пример:
- `25.04.2026 12:30`

Важно: время публикации контролируете вы, потому что публикация выполняется вручную.

### Для владельца

- `/add_admin <user_id>` — добавить/обновить админа
- `/remove_admin <user_id>` — удалить админа

## Требования

- Python **3.10+** (рекомендуется 3.11)
- Аккаунт Telegram и созданный канал
- Бот должен быть **админом канала**
- Включены **сообщения каналу** (Channel Direct Messages) в настройках канала

> Примечание для Windows: для работы таймзон через `zoneinfo` может понадобиться пакет `tzdata` (он добавлен в `requirements.txt`).

## Установка

1) Склонируйте/скачайте проект.

2) Создайте виртуальное окружение и установите зависимости:

- Windows (PowerShell):
  - `python -m venv .venv`
  - `./.venv/Scripts/Activate.ps1`
  - `pip install -r requirements.txt`

3) Настройте `config.py`:

- `BOT_TOKEN` — токен бота из @BotFather
- `CHANNEL_ID` — `@username` канала **или** числовой `chat_id`
- `OWNER_ID` — ваш `user_id` (как узнать — ниже)
- `DB_PATH` — путь к SQLite файлу (по умолчанию `volleyball_bot.db`)

Важно: не храните реальный `BOT_TOKEN` в публичном репозитории.

## Как узнать свой OWNER_ID

1) Поставьте в `config.py` значение `OWNER_ID = 0`.
2) Запустите бота.
3) Напишите боту в личку команду `/my_id`.
4) Впишите получившийся id в `OWNER_ID` и перезапустите бота.

При старте бот автоматически добавляет `OWNER_ID` в таблицу `admins` с ролью `owner`.

## Запуск

- `python bot.py`

После запуска:
- владелец может добавить админов командами `/add_admin`.
- админы управляют записью из личного чата.
- пользователи записываются через сообщения каналу.

## Сценарий использования

1) Админ: `/new_training` → вводит дату, время, лимит, уровень, **место проведения** → вводит время публикации (`сразу` или `ДД.ММ.ГГГГ ЧЧ:ММ`) → пишет `Да`.
2) Если выбрано `сразу` — бот публикует пост в канал немедленно.
3) Если указано время — бот запланирует публикацию и отправит пост в канал, когда наступит это время (MSK).
4) Пользователь пишет в сообщения каналу `Секция Фамилия Имя ...`.
5) Если места есть — статус **основной список** (active), иначе — **лист ожидания** (waiting).
6) Пользователь отправляет `Отмена` — запись снимается, первый из ожидания переводится в основной список и получает уведомление.

## Данные и структура БД

SQLite-файл: `volleyball_bot.db` (или `DB_PATH`).

Таблицы:
- `admins` — админы и владелец
- `trainings` — тренировки (open/closed) + поля для публикации (`publish_at`, `publish_status`)
- `registrations` — записи пользователей (active/waiting/cancelled)

## Частые проблемы

- **Бот не может писать в канал**: проверьте, что бот добавлен в канал и имеет право публиковать сообщения.
- **Пользователи не могут писать боту через канал**: проверьте, что в канале включены «сообщения каналу» (Channel Direct Messages).
- **Не работает `CHANNEL_ID`**: попробуйте указать числовой `chat_id` канала.

## Безопасность

- Не коммитьте `config.py` с реальным `BOT_TOKEN` в публичный репозиторий.
- При утечке токена немедленно перевыпустите его через @BotFather.
